# 上线前审查

## 已完成

- 已新增微信小程序项目结构：`project.config.json`、`miniprogram/app.json`、`miniprogram/app.wxss`、`miniprogram/pages/index/*`。
- 已把 Web 版核心能力迁移到小程序：轻量输入、添加作品、补充证据、生成作品名片、完整度建议、保存草稿、恢复草稿、复制分享文案。
- 已按小程序限制调整外链处理：作品链接不直接跳转，改为复制链接。
- 已做静态检查：`index.js` 语法通过，未发现 `window`、`document`、`localStorage`、`href` 等 Web-only 写法。
- 已移除小程序端兼容性较弱的 `display: grid` 和 `place-items`。

## 上线前必须处理

1. 替换真实 AppID
   - 当前 `project.config.json` 使用 `touristappid`，只能用于开发预览。
   - 正式提交审核前需要替换为你的真实微信小程序 AppID。

2. 绑定小程序类目和服务说明
   - 这个产品更接近“工具 / 效率 / 简历作品展示”类应用。
   - 具体类目需要在微信公众平台后台选择，避免审核时被认为服务内容不一致。

3. 补充隐私协议
   - 当前版本使用本地缓存保存草稿，不上传云端。
   - 如果后续增加登录、云存储、AI 接口或收集联系方式，需要补充隐私协议和用户授权说明。

4. 真机预览验证
   - 本机微信开发者工具 CLI 预览命令超时，未生成二维码。
   - 建议用微信开发者工具打开仓库根目录，使用真实 AppID 做一次编译和真机预览。

5. 外部链接策略
   - 小程序无法直接打开任意外部链接。
   - 当前版本采用复制链接；如果要内嵌打开链接，需要配置业务域名并使用 `web-view`，审核和域名要求会更高。

## 当前已知限制

- 没有账号体系，草稿只保存在本机微信缓存。
- 没有云端发布链接，生成的作品名片只能在小程序内查看或复制文案。
- 没有接入真实 AI，当前仍是规则提炼和结构化展示。
- 没有图片/文件上传能力，证明材料以文字和链接为主。
