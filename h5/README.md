# H5 作品名片

这是面向国内移动端用户的 H5 发布版。用户填写资料后发布到后端，获得一个 `/c/短ID` 访问链接；别人通过链接或二维码打开时，只看到对应用户的作品名片。

## 文件

- `index.html`: H5 前端页面，包含编辑器、作品名片预览、草稿保存、发布、短链接分享和二维码展示。
- `../server/server.js`: Node.js 后端，提供名片保存、读取、更新和短链接跳转。

## 本地预览

在仓库根目录运行：

```bash
npm start
```

然后访问 `http://localhost:8787/h5/index.html`。

## 部署建议

第二阶段需要后端存储，不建议只部署到纯静态资源服务。建议使用：

- 腾讯云 CloudBase / 云托管
- 阿里云轻量应用服务器
- 腾讯云 CVM + Nginx + Node.js
- 其他支持 Node.js 常驻服务的国内平台

如果部署到国内服务器或 CDN，需要按服务商要求完成域名备案，并开启 HTTPS。

## 注意

- 当前后端使用本地 JSON 文件保存数据，适合 MVP 和小规模验证；正式商业化建议换成云数据库。
- 发布后生成 `/c/短ID` 链接，不同用户会得到不同 ID，分享出去就是自己的作品名片。
- 二维码当前使用公开二维码服务生成，正式商业化建议换成自有后端或部署本地二维码库，避免国内访问不稳定。
- 当前页面不依赖海外字体或海外 JS CDN。
- 当前还没有账号体系，用户只能在原浏览器中保留编辑权限；后续可增加登录、作品管理和访问统计。
