# 上线前审查报告

审查对象：作品名片 H5 第二阶段版本。

审查结论：当前版本可以作为 MVP 部署到腾讯云服务器进行小规模真实用户验证。它已经具备“填写信息、发布到后端、生成短链接、他人打开只看名片、二维码分享”的完整闭环。

## 产品与交互

通过项：

- 编辑页和访问页已经区分：普通进入是生成工具，短链接进入是名片展示页。
- 分享出去的 `/c/短ID` 会读取后端数据，不再依赖超长 `#card=` 链接。
- 访问页不显示编辑器，符合“分享的是作品名片，而不是工具”的预期。
- 发布后会打开二维码区域，并提供复制访问链接和分享文案。
- 发布按钮增加了发布中状态，避免连续点击造成重复请求。
- 编辑页增加隐私提醒，降低用户误填敏感信息的风险。

仍需后续增强：

- 增加账号体系后，用户才能跨设备管理和修改自己的名片。
- 增加删除名片、投诉入口和内容审核，适合正式公开投放。
- 增加访问统计面板，后续可以作为商业化能力。

## UI 布局

通过项：

- 手机首屏没有横向溢出，标题和按钮在 390px 宽度下可正常显示。
- 访问页首屏优先展示头像、姓名、身份、简介、能力标签和联系方式，信息层级清晰。
- 编辑页保持轻量填写，基础信息在前，作品证据放在可选折叠项里，符合降低填写门槛的产品方向。
- 访问模式顶部只保留“分享”，不会干扰阅读。

建议：

- 后续可以增加模板选择，但第一版不要放太多样式选项，避免重新变重。
- 移动端可以再加一个底部固定“发布”按钮，提高长表单填写后的转化。

## 代码与安全

已修复风险：

- 静态服务现在只暴露 `h5/` 目录，避免访问到 `server/data/cards.json`、源码和小程序配置。
- 二维码在线部署时改为后端本地生成，不再依赖海外公开二维码服务。
- API 增加基础限流，降低被频繁发布刷爆本地 JSON 文件的风险。
- 响应增加基础安全头：`X-Content-Type-Options`、`X-Frame-Options`、`Referrer-Policy`、`Content-Security-Policy`。
- 用户输入会在后端做长度限制和 URL 协议过滤，前端渲染也做 HTML 转义。
- 生产部署支持 `BASE_URL` 和 `DATA_DIR`，便于腾讯云反向代理和持久化数据。

当前边界：

- JSON 文件存储适合 MVP，不适合高并发。
- 没有登录和实名能力，编辑权限只保存在发布者当前浏览器本地。
- 没有内容审核，公开投放前建议至少加敏感词过滤和投诉入口。

- 通过 Nginx 配置 HTTPS。
- 服务器防火墙只开放 `80`、`443` 和必要的 `22`。
- 配置 `./data/cards.json` 定时备份。
- 用手机微信访问正式域名，完整测试“填写、发布、复制链接、扫码、短链接访问”。
